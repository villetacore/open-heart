# Архитектура OpenHeart

Как устроена игра изнутри: слои, потоки данных, жизненный цикл, ключевые решения и их
причины. Дополняет [INDEX.md](INDEX.md) (что где лежит) взглядом «как это работает вместе».

## 1. Три слоя

```
┌────────────────────────────────────────────────────────────┐
│ Godot 4.7 (Forward Mobile / Vulkan)                        │
│ окно, рендер, физика, ввод, аудио-шина, ресурсы            │
│   main_menu.tscn ──▶ main.tscn (Game3D + Player)           │
└──────────────────────────┬─────────────────────────────────┘
                           │ GDExtension (OpenHeart.gdextension)
┌──────────────────────────▼─────────────────────────────────┐
│ openheart.dll — ВСЯ логика (Rust, gdext)                   │
│  Game3D: режимы, боёвка, HUD, порталы, квесты, миникарта   │
│  Player/Enemy: контроллер и AI (CharacterBody3D)           │
│  map/dungeon/world: построение геометрии                   │
└──────────────────────────┬─────────────────────────────────┘
                           │ serde_json (при старте сцены)
┌──────────────────────────▼─────────────────────────────────┐
│ godot/presets/<активный>/ — ДАННЫЕ = КОНТЕНТ = «игра»      │
│  weapons/classes/perks/enemies/items/npcs/quests/maps      │
└────────────────────────────────────────────────────────────┘
```

Редактор-плагин (`addons/oh_editor`, GDScript) живёт только в редакторе Godot и пишет
в нижний слой (JSON пресетов). В собранную игру не попадает.

## 2. Пресеты: «данные = игра»

- `content::discover_presets()` ищет папки в `res://presets/` **и** `user://presets/`
  (второе — для модов без пересборки).
- Активный пресет хранится в `Settings.preset`; **сейв содержит свой** `preset` —
  «Продолжить» всегда грузит ту игру, которую начали, независимо от настройки меню.
- Загрузчики контента (weapon/classes/perk) — **перезагружаемые**:
  `static RwLock<Option<&'static [T]>>` + `Box::leak` при загрузке. Это сохраняет
  повсеместный `&'static`-API и позволяет менять пресет в рантайме; утечка на смену
  пресета — единицы КБ (осознанный трейд-офф, задокументирован в `weapon.rs`).
- Каждый загрузчик при битом JSON пишет `godot_warn!` и падает на **встроенную копию**
  (`include_str!` из `presets/core/…`) — опечатка контент-мейкера не роняет игру.

## 3. Жизненный цикл

### Старт игры (`Game3D::ready`, порядок важен)

```
Settings::load → save::load (даёт preset приоритетно из сейва)
→ content::load_preset(preset)        # оружие/классы/перки
→ GameConfig::load_from(preset_base)  # враги/предметы/NPC/квесты/level
→ map::load_map(base, "hub")
     есть → build_map (геометрия+env+спавны из карты), gate из карты
     нет  → world::build_world (legacy-код) + спавны из level.json
→ build_npcs (npcs.json; при отсутствии ФАЙЛА — legacy NPC_DATA)
→ build_hud (+post-fx слой) → build_select_ui
→ есть класс в сейве? apply_loadout+Explore : ClassSelect
```

### Кадр (`process`)

Тики (флэши, NPC-анимация, FX-пулы, снаряды, анимация оружия, muzzle) →
`match mode` (ввод текущего режима) → обновление HUD (HP/XP/патроны/таргет/миникарта).
Стрельба и передвижение врагов — в `physics_process` соответствующих нод.

### Данж-цикл

```
Врата (E) → dungeon::generate(depth, seed++) → root на DUNGEON_OFFSET (500,0,500)
→ спавны из плана → телепорт игрока → миникарта из plan.floor_map
Босс убит → boss_alive=false → портал «глубже» активен (depth+1)
Выход → clear_dungeon (free root, срез врагов по x>250) → телепорт к вратам
Смерть → респавн в хабе, −25% золота, данж сгорает, сейв ЖИВ
```

Мир и данж сосуществуют в одном дереве сцены, разнесённые в пространстве
(`DUNGEON_OFFSET`); туман и дальность камеры скрывают «соседа» — без стриминга сцен.

## 4. Боевая модель

- **Hitscan**: луч из глаз через `PhysicsDirectSpaceState3D::intersect_ray`
  (стены честно блокируют), пеллеты дробовика — N лучей с разбросом в плоскости взгляда.
- **Мили**: выбор лучшей цели по углу+дистанции; вампиризм считается от **фактически
  нанесённого** урона (после резистов).
- **Снаряды**: пул `Projectile` (нода+позиция+скорость), сегментный ray-тест за кадр;
  сплэш — `explode()` с падением урона по радиусу и мягким самоуроном (×0.35).
- **Урон**: `дилт = amount × (1 − resist[dmg_type])`; резисты могут быть отрицательными
  (уязвимость). Типы: Physical/Fire/Energy/Void.
- **Прокачка**: XP за килы (`xp × mult` врага) → уровни (+очко перка, +HP/урон в
  `compute_loadout`) → перки/синергии (`perk::mods_for`) вливаются в `Loadout`.
- **Квесты**: `bump_quests(kind, target)` из точек событий (кил/подбор); сдача — в
  автогенерируемой сцене гивера (`Effect::QuestDone/Xp/Gold` через общий `apply`).

## 5. Карты и геометрия

`map.rs` строит из JSON: примитивы — реальные меши **с коллизией** (Box/CylinderShape).
**Рампы** — повёрнутый бокс: `pitch = atan2(Δy, длина)`, персонаж заходит по ним как по
склону (floor_max_angle 45°). Лестницы — N боксов. Земля и пол данжа режутся **полосами/
плитками** — это обходит лимит источников света на один меш у мобильного/совместимого
рендереров. Здания декларативны: бокс+вывеска+свет одной записью.

Данж (`dungeon.rs`, переработан в 0.4.0): комнаты с высотой пола 0/0.8/1.6 и своей
высотой потолка; коридоры наследуют высоты; на границах — ступени; `floor_map`
уходит в миникарту HUD.

## 6. Рендер и вид

- Рендерер: **Forward Mobile (Vulkan)** — держит glow/bloom и ACES-тонмаппинг
  (`build_environment`). Ambient 1.15, туман по карте.
- Неон: emission-плиты (`make_glow_slab`) + unshaded-спрайты вывесок → «светятся» в glow.
- Пиксель-арт: NEAREST(+mipmaps anisotropic на мешах); UI — canvas_items-стретч от
  1920×1080, поэтому хит-тест кликов меню совпадает на любом окне.
- Постобработка: полноэкранный `ColorRect` с canvas-шейдером (виньетка+CA+сканлайны)
  на CanvasLayer **под** HUD; `mouse_filter=IGNORE`, чтобы не глотать клики.

## 7. Сейвы и настройки

`SaveData` v2 (`user://save.json`): статы VN, RPG-прогрессия (класс/спек/уровень/XP/перки/
очки/сердца), арсенал (слоты+патроны+текущее), квест-прогресс (`quest_kills`), `preset`,
сид данжа. Все новые поля — `#[serde(default)]`: старые сейвы читаются. Настройки —
отдельно (`user://settings.json`).

## 8. Известные ограничения и их причины

| Ограничение | Причина / план |
|---|---|
| `game.rs` ~3 070 строк | Исторически; план M0 — разнести на системы + EventBus |
| Диалоги story-NPC захардкожены (`story.rs`) | VN-наследие; план §12 — dialogues/*.json |
| Смена пресета требует перезапуска сцены | Загрузчики перезагружаемы, но мир строится в `ready`; меню и так перезапускает сцену |
| Враги в МИРЕ ходят напрямик | В данже — A* по floor_map (nav.rs); карты мира не экспортируют сетку проходимости, там move_and_slide |
| `Box::leak` при смене пресета | Осознанная утечка ~КБ ради `&'static`-API |
| Editor-плагин пишет в `res://presets` | В dev-режиме res:// = папка проекта; для игроков-моддеров — `user://presets` |

## 9. Соглашения

- Новые поля контента — всегда `#[serde(default)]` + дефолт-функция: старые пресеты
  не ломаются.
- Тексты для игрока — `*_ru` (en — через `locale.rs`, пока только HUD/меню).
- Координаты карт: X вправо, Z вниз (юг), Y вверх; `pos` NPC — `[x, z]` (y всегда пол).
- Один коммит = одна фича, собранная и прогнанная по смоук-чеклисту
  ([CONTRIBUTING.md](../CONTRIBUTING.md)).
