# OpenHeart — Дизайн-план v2: «Большой мир»

Дата: 2026-07-04 · Статус: план к исполнению
Предыдущий этап (готово): костяк — 3 класса × 3 спека, 8 стволов с FP-спрайтами,
хаб + пустоши из кода, процедурный данж, HUD, диалоги/квесты в Rust-коде.

---

## 0. Цели этапа и принципы

**Цели:**
1. Мир и данжи — **авторские большие красивые карты**, а не «массив боксов из кода».
2. **Всё в конфигах**: враги, оружие, классы, перки, квесты, диалоги, лут, темы данжей, карты, звук, UI-тема.
3. Полноценное **RPG-ядро**: дерево перков + комбинации, боёвка со статусами и типами урона, 15+ врагов со способностями, элиты, боссы с фазами.
4. **Красота**: свет, шейдеры, постобработка, партиклы, анимации, звук, стилизованный UI.
5. **Техкачество**: чистые ассеты (пере-нарезка с исправлением дефектов), валидация данных, тесты, профилирование.

**Принципы:**
- Data-driven: код = механики, данные = контент. Добавление врага/оружия/квеста не требует правки Rust.
- Вертикальные срезы: каждый милстоун завершается играбельным билдом.
- Карты авторятся в **Tiled** (бесплатный редактор), данные — в JSON. Godot-редактор для контента не обязателен.
- Детерминизм генерации: сид → всегда одинаковый данж (тестируемо).

## 1. Архитектура и рефакторинг ядра (M0)

### 1.1 Проблема
`game.rs` — god-object на ~2300 строк: мир, боёвка, HUD, диалоги, порталы, сейвы в одном файле.
Оружие/классы/диалоги захардкожены в Rust (`weapon.rs`, `classes.rs`, `story.rs`).

### 1.2 Workspace из двух крейтов
```
rust/
├─ oh_core/     # чистая логика, БЕЗ Godot: конфиги+валидация, статы и формулы,
│               # генератор данжей, квест-машина, диалог-машина, лут-роллы.
│               # Здесь живут юнит-тесты (cargo test без движка).
└─ oh_game/     # gdext-слой: ноды, рендер, ввод, звук, UI. Тонкая обёртка над core.
```

### 1.3 Разбиение oh_game на системы
```
app.rs        — конечный автомат приложения: Menu → ClassSelect → Play → Dead
world_sys.rs  — загрузка/построение карт (Tiled), чанки, стриминг
dungeon_sys.rs— рантайм данжа (вход/выход/минимапа)
combat_sys.rs — выстрелы, снаряды, мили, статусы, урон
spawn_sys.rs  — спавн врагов/лута по таблицам, директор сложности
fx_sys.rs     — партиклы, декали, тряска камеры, hitstop (пулы объектов!)
hud_sys.rs    — HUD; ui/ — экраны (инвентарь, перки, журнал, карта, настройки)
dialogue_sys.rs, quest_sys.rs, audio_sys.rs, save_sys.rs, input_sys.rs
```

### 1.4 EventBus
Единая очередь событий кадра — системы не зовут друг друга напрямую:
```rust
enum GameEvent {
    EnemyKilled { id, kind, pos, killer_weapon },
    PlayerDamaged { amount, source }, ItemPicked { id },
    QuestAdvanced { quest, stage }, LevelUp { level },
    DungeonCleared { depth }, DoorOpened { id }, ...
}
```
Квесты слушают убийства, звук слушает всё, UI слушает всё. Убирает связность.

### 1.5 ContentDb
- При старте грузит весь `data/**`, отдаёт типизированные реестры.
- **Строгая валидация**: битый JSON → понятная ошибка (файл/поле); ссылочная целостность
  (quest → dialogue id → npc id, weapon → ammo type, enemy → ability id, tile gid → текстура).
- Горячая перезагрузка (F9 в debug): перечитать данные без перезапуска игры.

### 1.6 Инструменты отладки
Debug-консоль (~): `give <item>`, `spawn <enemy>`, `tp <x> <z>`, `god`, `noclip`,
`seed <n>`, `quest set <id> <stage>`, `killall`. Freecam, отрисовка коллизий и навигации.

## 3. Ассет-пайплайн v2 (M1) — «нормально парсить тексты и спрайты»

### 3.1 Аудит текущих дефектов (что чинить)
| Дефект | Где | Решение |
|---|---|---|
| Розовые подписи внутри спрайтов | `furn_bed` («МЕБЕЛЬ»), часть тайлов | label-mask по цвету при нарезке |
| Мусорные тайлы (смазанные кадры «световых схем») | `dtile_11, 23, 56` | QA-фильтр по энтропии + whitelist |
| Тайлы-«не текстуры» (рамки, RP-предметы) | `dtile_15–17, 57–64` (проверить) | ручной аудит → теги в манифесте |
| Слипшиеся объекты | улица (нет части объектов) | до-нарезка по manifests, не эвристикам |
| «Парение/утопание» персонажей | враги/NPC (подобрано константами) | якорь по нижней непрозрачной строке → `anchor_y` в манифест |
| Нет кадров attack/pain/death у врагов | все enemy_*.png (4 кадра) | генерация новых листов по шаблону SPRITE_SPEC 1024×640 |
| Неиспользуемые старые листы | `femboy_*.png` | конвертировать в новый формат или удалить |
| Ненарезанные секции атласов | мастер-атлас: декорации, ключи, детали стен; 18_08: одежда/декор комнат | дорезать по манифестам (нужны для интерьеров) |

### 3.2 Слайсер v3 — manifest-first
- `tools/manifests/*.json` описывают каждый атлас: секции, сетки, имена, спец-обработка.
  Нарезка идёт **строго по манифесту**; эвристики (детект полос) остаются только для первичной разметки.
- **Label-mask**: детект текста подписей по цветовому диапазону (розовый #ff7ab8±) в верхней зоне ячейки → затирание до удаления фона.
- **Якорь ног**: нижняя непрозрачная строка спрайта → `sprites_manifest.json: { anchor_y, trim, size }`;
  движок ставит биллборды по якорю — конец подбору констант.
- **QA-скрипт** (`tools/asset_qa.py`): alpha-покрытие, «пустые»/мусорные тайлы (энтропия, дисперсия),
  дубликаты (perceptual hash), отчёт-контактник HTML. Запуск в CI.
- Выход: `tiles.json` (id → файл, теги: wall/floor/ceiling/accent/hazard, материал шагов),
  `props.json` (id → файл, размер в метрах, коллизия, свечение).

### 3.3 Постобработка и стилизация текстур
- **Палитра проекта** (~28 цветов, розово-фиолетово-чёрная): мягкая квантизация всех текстур
  с лёгким dithering — визуальное единство трёх разных атласов.
- Батч: уровни/контраст, unsharp, **seamless-обработка** тайлов (offset+patch).
- **Normal map** из luminance (sobel) для стен/полов — объём при динамическом свете.
- **Emission map**: маска неоновых пикселей по цвету → неон реально светится (+glow в M6).
- Материал-фабрика в коде: albedo + normal + emission из манифеста автоматически.
- UI-иконки: апскейл ×2 (xBRZ) — чёткость на HUD.

## 4. Карты: мир и главная локация (M2)

### 4.1 Пайплайн Tiled → игра
Карты авторятся в **Tiled** (формат `.tmj` = JSON), парсер в `oh_core`, билдер мешей в `oh_game`:
- **Tile-слои**: `floor`, `elevation` (террасы/высоты), `walls` (высота из свойств тайла), `ceiling` (опц., для интерьеров).
- **Object-слои**: `props` (пропсы/префабы), `lights` (цвет/энергия/радиус/флика), `spawns`
  (enemy/npc/item/ammo/weapon + таблицы), `triggers` (зоны: квест, смена музыки, вход в интерьер, текст-подсказка), `decals` (граффити/постеры на стены).
- `tilesets.json`: gid → текстура + теги (материал, коллизия, hazard). `prefabs.json`:
  композиции (меш+спрайт+свет+коллизия) — «фонарь», «прилавок», «врата» описываются один раз.
- Геометрия: жадное слияние полос (как сейчас) + **чанки 16×16 тайлов** → куллинг по дистанции,
  MultiMesh для скаттера (мусор, трава, камешки).
- Рампы/лестницы: тайл-свойство `ramp` → наклонный меш; смена высот делает карты объёмными.

### 4.2 Главная локация — хаб «Неоновый квартал» (одна большая карта + интерьеры)
Районы: **Площадь Сердца** (фонтан, доска заданий), **Рынок** (торговец, магазин),
**Клуб FEMBOY CLUB** (интерьер: музыка, NPC, бар), **Медпункт** (лечение/респек за золото),
**Тёмная аллея** (граффити, секрет, Незнакомец), **Врата данжа**, 3 выхода в открытый мир.
Интерьеры — отдельные маленькие `.tmj` с дверьми-триггерами. 8 NPC получают посты, 2–3 — патрульные маршруты. Декор — мебель/ванная/неон из уже нарезанных пропсов + до-нарезка интерьерных секций атласов.

### 4.3 Открытый мир v1
- Размер: **~300×300 тайлов (900×900 м)**, 4 региона со своими тайлсетами, светом, туманом, спавн-таблицами и амбиентом:
  1. **Пустоши** (старт, ур. 1–3), 2. **Руины Проспекта** (ур. 3–6),
  3. **Кислотные Сады** (ур. 6–9, hazard-лужи), 4. **Некрополь Неона** (ур. 9+, поздний).
- **POI** в каждом регионе (3–5): лагеря врагов, входы мини-данжей, шрайны-баффы, странствующий торговец, секреты, событийные арены.
- Level-гейтинг мягкий: шкала опасности региона на карте мира.
- Карта мира UI — стилизованная (ассет `map` из атласа как основа), маркеры квестов/POI, fog по посещённому.

### 4.4 Навигация и производительность
- Враги: **A\* по тайл-сетке** (мир и данж единообразно) + separation steering, чтобы не слипались.
- Бюджет: 60 fps, ≤150 draw calls на сцену, ≤6 активных источников света на чанк.

## 5. Данжи 2.0 (M3)

- **Гибридная генерация**: процедурный граф комнат (комнаты + циклы) → каждая комната =
  **авторский room-template** из Tiled (`dungeons/rooms/*.tmj`, 8×8…16×16, с маркерами спавнов/лута/дверей) → коридоры-шаблоны → пост-декор по тегам темы.
- Фичи: **двери и цветные ключи** (спрайты ключей уже в атласе), **ловушки** (шипы, лава, пресс),
  **секретные стены** (сдвиг по E, подсветка при перке «Чутьё»), событ十е комнаты:
  **арена** (запечатка + волны), **сокровищница**, **алтарь перка** (временный бафф на забег), **комната-магазин**.
- **Боссы**: 1 на тему (3 шт), фазы по HP (смена паттернов способностей), своя арена-шаблон.
- **Модификаторы данжа** (глубина 3+): «Кровавый» (+враги +лут), «Тёмный» (мало света, +души), «Проклятый» (таймер до кары).
- **Минимапа** данжа с fog-of-war по посещённым тайлам.
- Награда первого прохождения глубины, лут-таблицы по глубине.

## 6. RPG-ядро: статы, классы, перки, дерево (M4)

### 6.1 Статы (расчёт в oh_core, всё модифицируемо перками)
`hp, armor, move_speed, dash_charges, crit_chance, crit_mult, attack_speed,
lifesteal, ammo_cap_mult, pickup_radius, luck` + резисты `{physical, fire, energy, void}`.
Формула: `урон = base × class_mult × перки × (крит?) × (1 − resist_цели) × модиф. статусов`.

### 6.2 Классы (из classes.json)
3 класса × 3 спека (как сейчас) + **активная классовая способность** (клавиша F, кд):
- Берсерк — «Ярость»: 6 с +lifesteal +скорость атаки;
- Штурмовик — «Штурм»: рывок-таран с уроном и стаггером;
- Оператор — «Овердрайв»: 4 с стрельба без расхода боезапаса.

### 6.3 Дерево перков и комбинации
- **Общее дерево** (3 ветки: Живучесть / Урон / Утилити) + **классовая ветка** (уникальная).
- Узлы: статы, механики («+1 заряд рывка», «взрыв бочек ×2»), триггеры («убийство в упор → +скорость на 3 с»), капстоуны веток.
- Очки: 1/уровень + за боссов и ключевые квесты. **Респек** у Медпункта за золото.
- **Синергии** (`synergies.json`): пара/тройка перков открывает именованный комбо-эффект;
  UI дерева показывает вкладку «Комбинации» с прогрессом («Вампиризм 2/2 + Скорость 1/2»).
- UI: полноэкранное дерево с зумом/скроллом, линии зависимостей, тултипы, превью изменения статов.

## 7. Боевая система «как в RPG» (M4–M5)

- **Типы урона**: Физический / Огонь / Энергия / Пустота. У врагов резисты/уязвимости → выбор оружия имеет смысл.
- **Статусы** (конфигурируются на оружии/способностях): Кровотечение (стакается), Горение (DoT),
  Замедление, Оглушение/Стаггер, Уязвимость (+входящий урон). Иконки статусов над врагом и на HUD игрока.
- **Хит-реакции**: pain-шанс по врагу (кадр pain + стаггер), knockback по массе, hitstop 40 мс на крите, тряска камеры.
- **Damage numbers** (всплывающие, крит — крупнее/цветом), hitmarker на прицеле, killfeed.
- **Враги со способностями** (`abilities.json`, компонуются): charge с телеграфом, веер снарядов,
  кислотный плевок (лужа-DoT), призыв миньонов, щит-фаза (пока жив тотем), телепорт-фланг,
  взрыв при смерти, лечение союзников, аура +урон. **Роли**: rusher / ranged / support / tank / swarm / caster / boss.
- **Элитные аффиксы** (`affixes.json`): Быстрый, Бронированный, Вампирический, Взрывной, Призрачный, Токсичный —
  элита = базовый враг + 1–2 аффикса + цветная подсветка. Цель: **15+ типов врагов** комбинаторикой роли×темы×аффиксы.
- **Директор сложности**: скейл от глубины данжа и уровня региона; волны в аренах.
- Апгрейды оружия («моды», 1–2 слота): «удушающий чок» дробовику, «зажигательные» автомату — из лута боссов.

## 8. Управление и физика (M5)

- **Движение**: ускорение/фрикшн-кривые, air-control ~30%, coyote time 0.1 с, jump-buffer 0.15 с,
  step-up 0.4 м (лестницы без прыжков), **рывок** (2 заряда, i-frames 0.2 с), **слайд** из спринта с сохранением скорости, headbob синхронно шагам, FOV 90→97 в спринте.
- **Отдача**: кик камеры по оружию, разброс автомата растёт при зажиме, восстановление при отпускании.
- **Физика**: слои/маски (world / player / enemy / player_proj / enemy_proj / pickup / trigger);
  гранаты — RigidBody с отскоком; взрыв даёт импульс телам; **взрывные бочки** (проп из атласа);
  пикапы дропаются с подпрыгиванием; разрушаемые ящики с лутом.
- Интеракция: единая клавиша E с контекстным промптом (двери/сундуки/рычаги/NPC/порталы/секретки).

## 9. Визуал: свет, шейдеры, постобработка, анимации (M6)

### 9.1 Рендерер — спайк в начале милстоуна
Тест **Forward+ (Vulkan)** на целевой машине (AMD Radeon): если стабильно → включаем
glow (неон засветится по emission-маскам из M1!), volumetric fog, SSAO, мягкие тени 2–3 ключевых источников.
Фолбэк — Compatibility + фейки: глоу-спрайты вокруг неона, дизеринг-туман, «god rays» биллбордами.

### 9.2 Шейдеры (пак `godot/shaders/`)
- **Пост**: color grading LUT (фирменный розово-фиолет), виньетка, лёгкие сканлайны/CRT (переключаемо),
  хроматическая аберрация при уроне, экранная ударная волна от взрывов, пульс + десатурация при низком HP.
- **Мир**: лава/жидкости (UV-scroll + distort + emission), мерцание неона (рандомизированная флика),
  портал (fresnel + вращение UV), **dissolve** смерти врагов (noise + светящаяся кромка),
  hit-flash белым (замена modulate), outline интерактивных объектов, height-fog у пола.
- **Партиклы** (GPUParticles3D, пулы): гильзы, искры попаданий, брызги крови + декаль-лужа,
  дым ствола, пепел/пылинки в воздухе (амбиент), светлячки у душ-факелов.

### 9.3 Анимации
- **Спрайт-аниматор** (стейт-машина в oh_core): idle/walk/attack/pain/death + события кадров
  («урон наносится на 2-м кадре атаки») — синхронизация геймплея и картинки.
- Новые листы врагов (M1) дают attack/pain/death; смерть = кадры + партикль + декаль.
- Камера: trauma-based тряска, наклон в стрейфе ~1.5°, FOV-кики, отдача.
- UI-твины: всплытия панелей, счётчики-роллеры, покачивание пикапов (sin), плавный XP-бар.

# 1. Собрать Rust и запустить одной командой (из корня проекта):

**Смоук-чеклист** (после любых изменений):
1. Меню открывается, «Новая игра» → экран выбора класса (1–3, затем спек 1–3).
2. Хаб рендерится: площадь, неон-вывески, NPC; WASD/мышь/Shift/Space работают, Esc отпускает мышь.

6. Босс убит → «портал вглубь открыт»; выход-портал возвращает в мир; сейв переживает рестарт.
