# Редактор игры (вкладка «OpenHeart»)

OpenHeart редактируется не правкой движка, а собственным редактором контента —
плагином внутри Godot. Философия: **фундамент закрыт, всё построенное в игре — открыто.**

## Открыть

1. Запусти редактор: `./run.ps1` (или `godot -e --path godot`).
2. Сверху, рядом с кнопками **2D / 3D / Script**, есть вкладка **OpenHeart** — жми её.
3. Если вкладки нет: Project → Project Settings → Plugins → включи «OpenHeart Editor».

## Устройство панели

```
┌ Пресет: [core ▾]  Карта: [hub.json ▾]  [id нового…][Создать копию]  [💾 Сохранить] [🔓 Ядро] [🎨 ИИ] ┐
├───────────────────────────┬─────────────────────────────────────────────────────────┤
│ Категория                 │  Форма выбранной записи                                 │
│  Оружие / Классы / Перки  │   id        [pyro_cultist        ]                      │
│  Синергии / Враги /       │   name      [Пиро-культист       ]                      │
│  Предметы / NPC / Квесты  │   hp        [77.0     ⟲]                                │
│  Диалоги / Пресет /       │   sprite    [cultist ▾]  ← скан enemy_*.png             │
│  Карта: блоки / здания /  │   pos       [12.0 ⟲][0.0 ⟲][−7.5 ⟲]                     │
│  свет / пропсы / флэты /  │   color     [■ палитра]                                 │
│  глоу-каналы / 4 спавна   │   resist    { "fire": 0.6, "void": -0.4 }   (JSON)      │
│ Записи   [🔎 поиск…]      │   …                                                     │
│  [pyro_cultist — Пиро…]▌  │                                                         │
│ [＋ Добавить][⧉ Дублировать][🗑 Удалить]                                             │
└───────────────────────────┴─────────────────────────────────────────────────────────┘
```

- **Категория** → список записей этого типа из JSON текущего пресета; поле **🔎 поиск**
  фильтрует записи по подстроке.
- **Карта: [… ▾]** — какой файл из `maps/` редактируют категории «Карта: …»
  (многокартные пресеты поддержаны данными; игра пока грузит `hub`).
- **Форма** генерируется по схеме: строки, числа, чекбоксы, векторы (спинбоксы),
  **цвета через палитру**, выпадающие списки — в том числе **динамические из данных
  пресета**: гивер квеста ← NPC, цель квеста ← враги/предметы, вид спавна ← враги/
  предметы/оружие, спрайт врага ← скан `enemy_*.png` (сгенерил в «🎨 ИИ-текстуры» —
  сразу в списке). Остальное вложенное — JSON-фрагмент с проверкой синтаксиса.
- Изменения живут в памяти панели до **«💾 Сохранить пресет»** (пишет все затронутые
  файлы). При сохранении выполняется **проверка ссылок** (квест → NPC/враг/предмет,
  спавны карт, next у диалогов) — проблемы не блокируют запись, но перечисляются
  в статусе и панели Output.
- Пресеты ищутся и в `res://presets`, и в `user://presets` (моды).
- Проверить в игре: F5 → изменения подхватываются при старте сцены (пересборка Rust не нужна).

## Пресеты: сделать «свою игру»

1. Введи id в поле «id нового пресета…» (латиница/цифры) → **«Создать копию»**.
   Текущий пресет целиком копируется в `godot/presets/<id>/`.
2. Правь что угодно: врагов, карту, классы, квесты — это уже отдельная игра.
3. В **главном меню игры** кнопка «Пресет: …» переключает между установленными;
   новая игра стартует в выбранном, сейв запоминает его.

Моддинг без исходников: игроки могут класть пресеты в `user://presets/<id>/`
(`%APPDATA%\Godot\app_userdata\OpenHeart\presets\` на Windows) — игра найдёт их сама.

## 🎨 ИИ-генерация текстур

Кнопка **«🎨 ИИ-текстуры»** в верхней панели открывает окно генерации ассетов
нейросетью — «в несколько кликов»:

1. Выбери **тип ассета**: персонаж/враг (512×256), тайловая текстура (512×512),
   иконка предмета (128×64) или FP-оружие (384×256).
2. Введи **id** (подсказка формата — в плейсхолдере: `enemy_*`, `wall_*`, `item_*`…)
   и **описание** — оно подставится в шаблон промпта типа (шаблоны:
   `tools/aigen_templates.json`, итоговый промпт виден тут же).
3. **«⚡ Сгенерировать»** — запрос уходит на сервер генерации, результат проходит
   постобработку (`tools/process_sprites.py`: удаление фона, ресайз, сборка кадров)
   и ложится сразу в `godot/assets/…`. Превью — в этом же окне; в игре ассет
   подхватится при следующем F5 (импорт Godot не нужен).

Сервер настраивается в **`tools/aigen.json`**: `url` — адрес сервера,
`backend` — `a1111` (AUTOMATIC1111 WebUI) или `comfyui`, плюс модель/шаги/сэмплер.
Редактор вызывает `python tools/aigen.py` (нужен Python с Pillow — `pip install Pillow`);
генерация идёт в фоне, редактор не блокируется. CLI работает и без редактора:

```bash
python tools/aigen.py character enemy_pyro "cultist in burning red robes"
python tools/aigen.py texture wall_lab "sci-fi metal panels, green glowing vents"
python tools/aigen.py item item_bomb "round black bomb with lit fuse" --seed 42
python tools/aigen.py character enemy_x --input got_from_chatgpt.png   # только постобработка
```

Сырые изображения до постобработки сохраняются в `tools/generated/` (в git не попадают).
Для нового врага не забудь указать его лист в `enemies.json`: поле `sprite` = имя
после `enemy_` (подсказку выводит и сам скрипт).

## 🔒 Замок ядра

Кнопка «Защитить ядро» ставит **read-only** (атрибут файловой системы) на фундаментальные
файлы: `main.tscn`, `main_menu.tscn`, `project.godot`, `OpenHeart.gdextension`.
В этом режиме ни редактор Godot, ни скрипты не перезапишут их случайно — контент-мейкер
физически не может «заменить фундаментальные элементы игры», работая при этом со всем
остальным без ограничений. Повторное нажатие снимает защиту (для разработчиков ядра).

Rust-код (`rust/`) в игру попадает только через пересборку DLL — из редактора он
недостижим по построению.

## Ограничения текущей версии

- Карта редактируется списком записей (без 3D-гизмо); проверяй расстановку запуском
  F5 — итерация быстрая, пересборка не нужна. 3D-превью — в планах
  ([DESIGN_PLAN.md](DESIGN_PLAN.md)).
- Игра пока загружает карту с id `hub`; остальные файлы из `maps/` редактируются,
  но в игру попадут после поддержки мультикарт движком.
- Диалоги редактируются полями lines/choices как JSON (граф-редактор — в планах);
  формат и эффекты — [DATA_FORMATS.md](DATA_FORMATS.md#dialoguesjson--сцены-диалогов-массив).
- Схемы полей заданы в `editor_main.gd::SCHEMAS` — расширяя формат данных, добавь поле
  и туда (см. [CONTRIBUTING.md](../CONTRIBUTING.md)).
- Кнопка защиты использует `attrib` — специфично для Windows (на других ОС просто
  не сработает, без вреда).
