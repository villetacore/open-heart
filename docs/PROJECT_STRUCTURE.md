# OpenHeart — структура проекта

Карта того, что где лежит и зачем. Актуально на v0.4.0 (2026-07-04). Если добавляешь
новые файлы — обнови и этот файл. Более детальный справочник (публичные API модулей,
счётчики, «где что искать») — в **[INDEX.md](INDEX.md)**.

## Как это всё связано (коротко)

```
Godot/Redot (движок, рендер, ввод, окно)
        │  грузит через OpenHeart.gdextension
        ▼
openheart.dll  ←── компилируется из rust/src/*.rs
        │  при старте читает
        ▼
godot/data/*.json  (оружие, классы, враги, перки, квесты...)
```

Правило: **логика — в Rust, контент — в JSON, сцены/ассеты — в Godot.** Новый враг или
оружие добавляется правкой JSON, а не кода (см. [DESIGN_PLAN §2](DESIGN_PLAN.md)).


## `rust/` — вся логика игры

```
rust/
├─ Cargo.toml, Cargo.lock   — зависимости: godot (gdext), serde, serde_json
├─ .cargo/config.toml       — настройки сборки (Redot LTS = Godot 4.3 API)
└─ src/
   ├─ lib.rs                — точка входа GDExtension, список модулей (mod ...)
   │
   │  ── Оркестрация игры ──
   ├─ game.rs               — ГЛАВНЫЙ файл (Game3D). Режимы игры (выбор класса → исследование →
   │                          диалог → инвентарь → перки → смерть), HUD целиком (+миникарта
   │                          данжа, пост-шейдер), боёвка (hitscan/мили/снаряды), порталы,
   │                          спавны, квест-раннер. Самый большой файл — по плану (M0)
   │                          со временем будет разбит на подсистемы.
   ├─ map.rs                 — карты из JSON: box/ramp/stairs/cylinder, здания, glow, спавны
   ├─ content.rs             — ContentDb: `load_preset(id)` читает данные пресета при старте
   │
   │  ── Data-driven контент (JSON + фолбэк) ──
   ├─ weapon.rs              — 8 оружий из weapons.json: урон, тип урона, боеприпасы, FP-спрайты
   ├─ classes.rs             — 3 класса × 3 спека из classes.json, расчёт итоговых статов (Loadout)
   ├─ perk.rs                — дерево перков + синергии из perks.json/synergies.json
   ├─ config.rs              — загрузчик enemies.json/items.json/level.json + Resist (резисты урона)
   │
   │  ── Игровые сущности (Godot-ноды) ──
   ├─ player.rs              — Player (CharacterBody3D): FPS-движение, мышь, прыжок, спринт
   ├─ enemy.rs               — Enemy (CharacterBody3D): AI патруль→погоня→атака, резисты, LOS-проверка
   ├─ world.rs               — открытый мир: хаб-площадь + пустоши, строится кодом (не Tiled — пока)
   ├─ dungeon.rs             — процедурный генератор данжей: комнаты на разных высотах,
   │                          переменные потолки, ступени, платформы, темы, босс, floor_map
   ├─ gfx.rs                 — утилиты рендера: боксы/биллборды/свет, TexCache, детерминированный RNG
   │
   │  ── Состояние и сохранение ──
   ├─ game_state.rs          — GameState: RPG-прогрессия (уровень/XP/класс/перки/золото/квесты/
   │                          инвентарь/сид данжа). Внутри же остались Period/Location/Action —
   │                          НАСЛЕДИЕ от прежнего прототипа визуальной новеллы, сейчас не
   │                          используются 3D-игрой напрямую (см. «Наследие VN» ниже).
   ├─ save.rs                — SaveData v2 ↔ user://save.json (сериализация всего GameState)
   ├─ settings.rs            — язык/громкость/чувствительность мыши ↔ user://settings.json
   │
   │  ── Диалоги/квесты (общая система) ──
   ├─ dialogue.rs             — структуры Scene/Line/Choice/Effect для диалогов
   ├─ story.rs                — САМИ диалоги (хардкод, ~1000 строк) — реплики NPC. Кандидат на
   │                           перенос в dialogues/*.json (см. DESIGN_PLAN §12, ещё не сделано)
   ├─ quest.rs                — Quest/QuestLog (активен/выполнен)
   ├─ item.rs                — Item/Inventory (расходники, ключи)
   ├─ npc.rs                  — вспомогательная структура NPC для диалоговой системы
   ├─ character.rs            — Stats (INT/CHR/FIT/REP/WIL) — статы для гейтинга реплик в диалогах
   │                           («нужен ИНТ 7») — тоже из VN-прототипа, не путать с RPG-статами
   │                           (HP/урон/скорость), которые считает classes.rs.
   ├─ locale.rs               — таблица строк ru/en для HUD и меню
   └─ main_menu.rs            — MainMenu (Control): главное меню, экран настроек
```

### Наследие визуальной новеллы (важно понимать)

Проект начинался как **визуальная новелла** (см. `README.md`, старые коммиты), и часть кода
оттуда осталась: `character.rs` (статы INT/CHR/FIT/REP/WIL), `game_state.rs` (Period/Location/
Action), весь `story.rs`. Диалоги NPC в хабе (Вейл, Виктор, Елена...) до сих пор используют
именно эту систему — она рабочая и нужная. Но она **не связана** с RPG-прогрессией
(уровень/XP/класс/перки), которую добавили позже, — это два независимых слоя в одном файле
`game_state.rs`. План — со временем свести диалоги на общую data-driven систему (DESIGN_PLAN §12),
но пока это не сделано, обе системы существуют параллельно.

## `tools/` — пайплайн подготовки ассетов

Всё это — Python-скрипты, которые режут большие атласы (сгенерированные через ChatGPT) на
отдельные игровые спрайты. Не часть игры, но нужны при добавлении новых ассетов.

**Актуальный пайплайн** (см. `tools/ASSET_GUIDE.md` для полного гайда по промптам генерации):
```
1. slice_atlases.py   — основная нарезка: оружие, пикапы, эффекты, UI, небо, тайлы данжа, пропсы
2. slice_fix.py        — правочный проход: убирает слипшиеся ячейки, подписи, доп. чистка
3. slice_props.py      — финальная точная нарезка пропсов по вручную измеренным координатам
```
Результат кладётся прямо в `godot/assets/...`. Контрольные листы для проверки нарезки пишутся
в `tools/preview/` — эта папка в `.gitignore`, регенерируется скриптами, в репозитории не хранится.

**Одноразовые/вспомогательные скрипты** (использовались при разработке, оставлены для справки):
`analyze_regions.py`, `analyze_weapons.py` — поиск координат секций в атласе по плотности пикселей.
`extract_atlas.py`, `process_sprites.py`, `debug_cells.py`, `find_boundaries.py`,
`fix_fire_sprite.py`, `scan_boundary.py`, `scan_tex_row.py` — более старые одноразовые инструменты
из самого начала пайплайна нарезки.

`ChatGPT Image 30 июн. 2026 г., 00_00_28.png` — исходный «мастер-атлас» (эффекты/UI/небо/жидкости),
из которого режут `slice_atlases.py`.

| Добавить перк или синергию | `godot/presets/<id>/perks.json` / `synergies.json` |

| Понять открытые вопросы по геймдизайну | `docs/OPEN_QUESTIONS.md` |
