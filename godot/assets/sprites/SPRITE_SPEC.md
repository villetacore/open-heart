# OpenHeart — Спецификация спрайт-листов

## Папки

```
godot/assets/
├── sprites_raw/   ← исходники от художника (RGB PNG, тёмный фон)
├── sprites/       ← конвертированные в RGBA (скрипт ниже), их читает игра
└── sprites/SPRITE_SPEC.md  ← этот файл
```

## Формат существующих листов (femboy_*.png)

Размер холста: **1536 × 1024 px**  
Фон оригинала: тёмный RGB (~9,14,15) → конвертируется в прозрачный RGBA.

### Сетка

Левая колонка (x = 0–133) — текстовые метки, **не спрайты**.  
Правая секция (x ≥ 1000) — справочные примеры, **не используются в игре**.

Основная зона: **x = 134–958**, 4 направления × 2 кадра = 8 ячеек.

| Направление | Кадр 0 (x начало) | Кадр 1 (x начало) | Ширина контента |
|-------------|-------------------|--------------------|-----------------|
| ВПЕРЁД      | 134               | 250                | ~54 px          |
| ВПРАВО      | 366               | 474                | ~53 px          |
| НАЗАД       | 584               | 691                | ~60 px          |
| ВЛЕВО       | 806               | 907                | ~51 px          |

### Строки анимаций (y-координаты)

| Анимация     | y начало | Высота | Кадров | FPS в игре | Когда играет               |
|--------------|----------|--------|--------|------------|----------------------------|
| IDLE         | 100      | 148    | 2×4    | 3          | NPC стоит                  |
| WALK         | 258      | 113    | 2×4    | 8          | NPC движется               |
| ATTACK       | 380      | 113    | 2×4    | 12         | Рукопашная атака           |
| ATTACK2      | 504      | 124    | 2×4    | 10         | Стрельба / дальняя атака   |
| PAN          | 628      | 106    | 2×4    | 4          | Диалог активен             |
| DEATH        | 734      | 246    | 4 (горизонт.) | 8   | HP = 0                     |

> DEATH особенная: 4 кадра идут горизонтально на всю ширину (~200 px каждый),
> персонаж падает на бок — ячейки не делятся по направлениям.

### Координаты Rect2 для кода (ВПЕРЁД, Rust)

// PAN (диалог)

## Стандарт для НОВЫХ листов

Если художник делает нового персонажа — использовать этот формат.  
Он чище, степень двойки, без текстовых меток.

### Размер холста: 1024 × 640 px

### Сетка ячеек: 128 × 128 px

```
Колонки (x):  0   128  256  384  512  640  768  896
               ↓    ↓    ↓    ↓    ↓    ↓    ↓    ↓
              [ВПЕРЁД f0][f1][ВПРАВО f0][f1][НАЗАД f0][f1][ВЛЕВО f0][f1]

Строки (y):
  y=  0 — IDLE
  y=128 — WALK
  y=256 — ATTACK
  y=384 — PAN / TALK
  y=512 — DEATH (4 кадра × 256 px по ширине, высота 128)
```

### Правила для художника

1. **Фон** — полностью прозрачный (`RGBA PNG`). Никакого тёмного фона.
2. **Персонаж в ячейке** — центр по X, ноги на нижней границе ячейки (y = 127).
3. **Высота персонажа** — не выше 110 px внутри ячейки (оставить 18 px запас сверху).
4. **DEATH** — персонаж горизонтально, 4 кадра по 256 px ширина, в строке y=512.
5. **Именование файлов** — `npc_{id}.png`, например `npc_vale.png`, `npc_elena.png`.
6. **Хранить оригинал** в `sprites_raw/`, игра читает из `sprites/`.

### В коде (pixel_size для новых листов)

```rust
const PIXEL_SZ: f32 = 0.012; // 128 px × 0.012 = 1.536 m высота персонажа
```

## Конвертация RGB → RGBA (скрипт PowerShell)

Запустить один раз при добавлении нового листа.  
Делает тёмный фон прозрачным (порог: R<30, G<30, B<30).

```powershell
Add-Type -AssemblyName System.Drawing
$src = "sprites_raw\npc_vale.png"
$dst = "sprites\npc_vale.png"
$bmp = [System.Drawing.Bitmap]::new($src)
$out = [System.Drawing.Bitmap]::new($bmp.Width, $bmp.Height, [System.Drawing.Imaging.PixelFormat]::Format32bppArgb)
for ($y = 0; $y -lt $bmp.Height; $y++) {
    for ($x = 0; $x -lt $bmp.Width; $x++) {
        $px = $bmp.GetPixel($x, $y)
        if ($px.R -lt 30 -and $px.G -lt 30 -and $px.B -lt 30) {
            $out.SetPixel($x, $y, [System.Drawing.Color]::Transparent)
        } else {
            $out.SetPixel($x, $y, $px)
        }
    }
}
$out.Save($dst, [System.Drawing.Imaging.ImageFormat]::Png)
$bmp.Dispose(); $out.Dispose()
Write-Host "Done: $dst"
```

## Подключение нового NPC в игру (Rust)

В `rust/src/game.rs`, массив `NPC_DATA`:

```rust
NpcCfg {
    id: "vale",
    name: "Ms. Вейл",
    scene_id: "meet_vale",
    pos: Vector3::new(-6.0, 0.0, -6.0),
    color: Color::from_rgba(1.0, 0.5, 0.7, 1.0),
    tex: "res://assets/sprites/npc_vale.png",
},
```

Анимация подхватится автоматически — `tick_npc_anim()` применяет IDLE-кадры ко всем спрайтам.
